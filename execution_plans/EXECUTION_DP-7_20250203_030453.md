## JIRA Breakdown: OAUTH 2 Authentication with Active Directory Integration

**Epic:** Add Authentication with OAUTH 2 and integrate with Enterprise Active Directory

**Epic Description:** Add Authentication with OAUTH 2 and integrate with Enterprise Active Directory. This includes support for both machine-to-machine and human-to-machine authentication. Human users will already have identities in Active Directory, so the solution should leverage OpenID Connect to source user details.

**Constraints:** Existing enterprise OAUTH 2 server implemented by PING Federate.

---

**User Story:**

**Title:** As a user, I want to authenticate using my existing Active Directory credentials so that I can access the application securely.

**Story Points:** 8

**Sub-task:**

* **Title:** Define mapping between Active Directory attributes and application user attributes.
* **Description:** Determine which Active Directory attributes (e.g., sAMAccountName, mail) correspond to the application's user identifier, email, etc. Document this mapping for configuration and implementation.
* **Story Points:** 3

---

**Technical Task:**

**Title:** Implement OAUTH 2 Authentication with PING Federate and OpenID Connect.

**Description:** Integrate the Python application with the existing PING Federate OAUTH 2 server using OpenID Connect. This involves:

* Configuring PING Federate as an OpenID Connect Provider (OP) for the application.
* Implementing an OpenID Connect Relying Party (RP) library in the Python application.
* Handling authentication requests and responses according to the OpenID Connect flow.
* Retrieving and validating user information from the ID Token provided by PING Federate.
* Mapping Active Directory attributes to application user attributes based on the defined mapping.

**Sub-task:**

* **Title:** Research and select a suitable Python OpenID Connect library.
* **Description:** Evaluate and choose a well-maintained and documented Python library that supports OpenID Connect and integrates well with the application framework.
* **Story Points:** 3

---

This breakdown provides a high-level structure for implementing the epic. Further refinement and additional tasks may be required during the development process.
